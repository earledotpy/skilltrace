1.3 Adaptive Sequencing Logic
The system does not advance on time. Advancement requires mastery threshold met on all inbound edges.

Remediation protocol:
If Bayesian Knowledge Tracing probability P(Know) < 0.75 for a prerequisite, the scheduler blocks the dependent node
System prescribes the shortest remediation path: 2 formative tasks from the failed node, then 1 interleaved task combining failed and current node
This implements the Knowledge Space Theory principle of prerequisite closure

PILLAR 3: COMPETENCY ASSESSMENT & VALIDATION ENGINE

3.1 Formative vs Summative
Formative: Daily 5-question quizzes (Anki with cloze deletion), autograded notebooks via nbgrader, pytest unit tests for functions
Summative: End-of-node capstone requiring (a) working code, (b) evaluation report, (c) 500-word technical analysis

3.2 Automated Grading
Local toolchain: Python 3.11, pytest, nbgrader, pre-commit hooks
Code playgrounds execute in isolated venv; no cloud dependency
For LLM tasks: automated metrics (perplexity, ROUGE-L, accuracy) computed locally against small validation sets

3.3 Mastery Threshold Logic
MVP implementation uses consecutive-correct heuristic for simplicity on constrained hardware:
Foundational nodes: 3 consecutive formative passes at ≥90%
Core nodes: 2 consecutive passes + 1 successful summative
Advanced implementation: Bayesian Knowledge Tracing with P(L0)=0.2, P(T)=0.25, P(G)=0.15, P(S)=0.1; mastery declared at P(Know) > 0.85

This avoids time-based progression and implements true competency gating.
